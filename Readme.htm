Installation Instructions

Installation of the MACM software requires several steps. If you are unfamiliar with web servers then get assistance from the system administrators for your campus. The software has been installed and test on Windows 2000 and XP running IIS and on Mac OS X running Apache.


Step 1: Install PHP

The MACM software requires that PHP be installed and running on the web server. PHP is open-source and freely available. PHP can be obtained from http://www.php.net/ and may already be installed on the server. Follow the instructions provided with the PHP distribution for your server environment.

**NOTE:If any of the admin's for the installation of MACM will be using MAC or UNIX to create new experiments you will need to use PHP 4.3 or higher and follow the directions below to adjust the necessary setting in the php.ini file to enable the servers ability to read MAC and UNIX text files. (*.txt files are used to upload new experiments)

  1. Open php.ini file. (location varies depending on installation platform. Check php manual for details on location. (ie: Windows XP location = "C:\windows\php.ini"))
  2. Within the php.ini file run a find for "auto_detect_line_endings"
  3. Change its value from "off" to "on"
  4. Save php.ini and close it.
  5. Restart the PHP server


Step 2: Install MySQL

The MACM software requires that MySQL be installed, running, and have an account setup for your use. MySQL is open-source and freely available. MySQL can be obtained from http://www.mysql.com/ and may already be installed for your server environment. If your are not installing this yourself, the system administrator will assign you a username and password for the MySQL environment.


Step 3: Unzip the distribution file

Unzip the distribution files. Do not change the relative locations of any of the files or the software will not work.


Step 4: Edit CAPM.php

In the "Connections" directory of the distribution there is a file called "CAPM.php". Using a text processor such as notepad on Windows or BBEdit on the Mac you need to edit three lines. DO NOT use a word processor such as MS Word to edit this file. The following lines to be edited are:

// HOST NAME OR SERVER ADDRESS
// Usually this is the name of your domain (www.yoursite.com).
// In some cases the hostname will instead be "localhost".
$hostname_CAPM = "localhost"; <<<-set this to your server address


// MySQL USERNAME
// This is the username you use to access your MySQL database.
// Note: This is usually NOT the same as your FTP username.
$username_CAPM = "myUser"; <<<- set this to the MySQL account username


// MySQL PASSWORD
// The password used to access your MySQL database.
// Note: This is usually NOT the same as your FTP password.
$password_CAPM = "myPassword"; <<<- set this to the MySQL account password

Save the changes to the file.


Step 5: Edit install_DB.php

In the root directory of the distribution there is a file called "install_DB.php". Using a text processor such as notepad on Windows or BBedit on the Mac you need to edit one line. DO NOT use a word processor such as MS Word to edit this file. The line to be edited in just before the end of the file. You need to set the username and password the MACM environment.

mysql_query("INSERT INTO users
(user_name,password,user_type,email,get_email,logins,experiment_id)
VALUES ('Admin','Admin','admin','N/A','no',0,1)", $CAPM) or die(mysql_error());

^^^ Edit the first two entries in this values statement. The first entry is the username you want as administrator, the second is the password. DO NOT change the third entry in this statement.

Save the changes to the file.


Step 6: Move the files to the server

Using FTP (or another method) move the files in the distribution to a dedicated directory on the server.


Step 7: Execute the install_DB.php script

Using a web browser execute the install script by connecting the address http://www.myserver.edu/myDir/install_DB.php which will setup the appropriate tables in the database. The "myserver" is the server address and "myDir" is the directory that the files were moved to in Step 6.


Step 8: Connect to the MACM environment

Using a web browser connect to the MACM environment at http://www.myserver.edu/myDir/ and login using the username and password you setup in Step 4.


Using the environment:

The default setup contains no experiments. After logging in the opening page be "create a new experiment." Follow the on-screen instructions to create a text file with the students accounts, passwords, and initial portfolios of assets.
